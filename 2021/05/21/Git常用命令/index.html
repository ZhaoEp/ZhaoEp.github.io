Git

常用命令

pull

将远程库中的更改合并到指定分支：git pull origin next:master

与当前分支合并：git pull origin next

存在 tracking 的分支合并：git pull origin

默认情况下，git pull = git fetch + git merge FETCH_HEAD

如果使用--rebase ，则 git pull = git fetch + git rebase FETCH_HEAD

push

将本地分支更新，推送到远端：git push origin master

如果master不存在，会新建。

config

名字：git config user.name "maxsu"

邮箱：git config user.email "yiibai.com@gmail.com"

配置远端仓库：git remote add name git@github.com:yourName/repositoryname.git

查看配置：git config --list

branch

创建分支：git branch name

显示远程和本地分支：git branch -a

切换分支：git checkout name

合并分支：git merge name

删除分支：git branch --delete name

建立tracking：git branch --set-upstream-to=origin/<branch> master

tracking：在某些场合，Git会自动在本地分支与远程分支之间，建立一种追踪关系(tracking)。比如，在git clone的时候，所有本地分支默认与远程主机的同名分支，建立追踪关系，也就是说，本地的master分支自动”追踪”origin/master分支。

其他

git diff：用于显示提交和工作树等之间的更改

git log：显示提交日志

git status：显示索引文件和当前HEAD提交之间的差异，通常在git commi之前使用，检查文件状态。

git stash：用于将更改存储在脏工作目录中。

问题

  1. pull的时候出现：fatal:refusing to merge unrelated histories

解决：git pull github main --allow-unrelated-histories

  1. push的时候，出现：failed to push some refs to ...

解决：提示的意思是，远端和当前分支不匹配，指定当前分支，远端会自动创建一个新的分支。或者将当前分支和远端分支进行手动的tracking：git branch --set-upstream-to=github/main master
